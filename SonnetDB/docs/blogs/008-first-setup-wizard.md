## 首次设置向导：从零开始配置 SonnetDB

当您首次通过浏览器访问 SonnetDB 管理界面时，系统会自动引导您进入首次设置向导（First Setup Wizard）。这个向导会帮助您完成管理员账户创建和安全令牌配置等关键步骤。本文将详细介绍整个设置过程。

### 访问管理界面

使用 Docker 部署或本地启动 SonnetDB 后，打开浏览器访问 `http://localhost:8840`，您将自动进入设置向导界面。如果 SonnetDB 尚未完成初始化，系统会显示一个欢迎页面，引导您开始配置。

界面语言会自动适配浏览器的语言设置，目前支持中文和英文，方便不同语言背景的用户使用。

### 第一步：创建管理员用户

向导的第一步是创建管理员账户。您需要填写以下信息：

- **用户名**：管理员账户的名称，例如 `admin`。
- **密码**：管理员密码，建议使用强密码（包含大小写字母、数字和特殊字符，长度不少于 8 位）。
- **确认密码**：再次输入密码以确认。

请注意，SonnetDB 默认不设默认密码，首次访问时必须设置。这是为了确保从初始状态开始就具备安全保障。

填写完毕后，点击"下一步"继续。

### 第二步：生成 Bearer Token

管理员用户创建成功后，向导会引导您生成一个用于 API 访问的 Bearer Token。这个令牌将用于应用程序与 SonnetDB 之间进行身份验证。

您可以配置令牌的以下属性：

- **令牌名称**：一个可识别的名称，例如 `my-app-token` 或 `ci-cd-token`，方便您在多个令牌中进行管理。
- **过期时间**：可以设置为永不过期，也可以设置一个具体的过期日期。对于生产环境，建议设置合理的过期时间并定期轮换。

令牌生成后，系统会显示完整的令牌字符串。**请务必立即复制并保存好这个令牌**——出于安全考虑，关闭此页面后将无法再次查看完整的令牌内容。如果您不慎丢失了令牌，可以在管理界面的"令牌管理"页面中撤销旧令牌并重新生成。

### 第三步：系统概览

完成上述步骤后，向导会显示一个系统概览页面，包含以下信息：

- 服务器名称和版本号
- 存储路径和数据目录
- 已创建的测量（measurement）数量
- 系统健康状态

从这里开始，您就正式进入了 SonnetDB 的管理控制台主界面。

### 管理控制台概览

完成设置后，您将看到以下主要功能区域：

1. **SQL 控制台**：这是最常用的功能，您可以直接在浏览器中编写和执行 SQL 语句，查看查询结果。
2. **数据监控**：显示数据库的运行状态，包括写入速率、查询速率、活跃连接数等实时指标。
3. **测量管理**：查看和管理所有已创建的 measurement，包括其结构和数据预览。
4. **用户与令牌管理**：管理用户账户、角色权限和 API 令牌。
5. **设置页面**：配置数据库参数、数据保留策略、日志级别等。

### 重置设置

如果您需要重新进行首次设置（例如部署了一个全新的实例），可以在服务端配置文件中删除已有的安全配置，然后重启服务。在 Docker 部署中，删除数据卷并重新创建容器也会触发重新初始化。

首次设置向导的设计初衷是降低用户的入门门槛。整个流程只需要不到两分钟即可完成，之后您就可以开始创建 measurement、写入数据并执行查询了。后续文章中，我们将详细介绍如何通过 SQL 控制台进行数据操作。
